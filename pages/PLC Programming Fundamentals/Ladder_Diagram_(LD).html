Ladder Diagram (LD)

The Ladder Diagram (LD) is one of the most widely used and fundamental PLC programming languages in industrial automation. It is a graphical programming language that represents control logic in a form similar to traditional electrical relay logic diagrams. Due to its simplicity, readability, and close resemblance to electrical schematics, Ladder Diagram is especially popular among electrical engineers and technicians.

Ladder Diagram programming is standardized under IEC 61131-3 and is commonly used for discrete control applications such as motor control, interlocking, sequencing, and safety logic.

Concept of Ladder Diagram

The Ladder Diagram is based on the concept of a ladder, where two vertical lines represent power rails and horizontal lines represent rungs. Each rung contains logical conditions and output actions that are evaluated by the PLC.

  • The left vertical line represents the power supply or logic TRUE condition.
  • The right vertical line represents the return path or logic FALSE condition.
  • The horizontal rungs contain input conditions and output instructions.

The PLC evaluates each rung from left to right and from top to bottom, similar to reading a ladder.

Basic Elements of Ladder Diagram

  • Contacts: Contacts represent input conditions or internal logic states in the PLC program.
    • Normally Open (NO) Contact: Represents an input that is FALSE when inactive and becomes TRUE when activated.
    • Normally Closed (NC) Contact: Represents an input that is TRUE when inactive and becomes FALSE when activated.
    Contacts do not physically open or close; they only represent the logical status of inputs, outputs, or internal memory bits.
  • Coils: Coils represent output actions in a Ladder Diagram.
    • When the logic on the left side of the rung becomes TRUE, the coil is energized.
    • Energizing a coil sets the corresponding output or internal memory bit to TRUE.
    Coils are used to control physical output devices such as motors, solenoid valves, relays, and indicators.

Working of Ladder Diagram

During the PLC scan cycle, the Ladder Diagram program is executed rung by rung.

  1. The PLC reads all input signals and stores their status in memory.
  2. The Ladder Diagram logic is executed using these stored input values.
  3. If all conditions in a rung are TRUE, the output coil at the end of the rung is energized.
  4. Output results are stored in the output image table and updated at the end of the scan cycle.

This process repeats continuously, allowing real-time control of industrial processes.

Common Ladder Diagram Instructions

Ladder Diagram supports various instructions used in automation logic, such as:

  • Logical instructions: AND, OR, NOT.
  • Timers: TON (On-Delay), TOFF (Off-Delay), RTO (Retentive Timer).
  • Counters: CTU, CTD, CTUD.
  • Set and Reset coils.
  • Comparison instructions: =, >,<
  • Move and arithmetic instructions.

Applications of Ladder Diagram

Ladder Diagram is commonly used in:

  • Motor control circuits (start/stop, interlocking).
  • Conveyor systems.
  • Packaging machines.
  • Safety circuits and emergency stop logic.
  • Basic machine automation and sequencing.
